# 문서의 목적

이 문서는 구현 요구사항의 수행 여부를 추적하기 위해 만들어졌습니다.
주요 구현 요구사항에 있는 모든 항목들이 완료될 때까지 이 문서에 각 작업의 완료 여부를 표시하며 진행하세요.

## 주요 구현 요구사항

## 1️⃣ Home

### ✅ 시작

- [x] 프로젝트가 실행되면 홈 화면을 보여주세요.

### ✅ 검색

- [x] 검색창의 `활성화(active)` 및 `호버(hover)` 상태에 따라 스타일을 변경해주세요.
- [x] 검색 기능을 구현해주세요. 검색 버튼이 따로 존재하지 않으므로, 키보드 이벤트를 적용해주세요.
  - 엔터키가 입력될 때 검색이 되도록 구현해주세요

## 2️⃣ Results

### ✅ 검색

- [x] 검색창의 `활성화(active)` 및 `호버(hover)` 상태에 따라 스타일을 변경해주세요.
- [x] 검색창의 입력값 유무에 따라 전체 삭제 아이콘을 추가해주세요. 클릭하면 입력값이 초기화 되어야 합니다.
  - 엔터키가 입력될 때 검색이 되도록 구현해주세요
- [x] 검색 결과를 받아오는 동안 스켈레톤을 보여주세요. 스켈레톤이 보여지는 동안에도 검색이 가능해야 합니다.
- [x] 에러 응답이 돌아올 수 있습니다. 피그마를 참고하셔서 에러 핸들링을 해주세요.

### ✅ 헤더

- [x] 뒤로 가기 기능을 구현해주세요.
- [x] 스크롤 시 헤더에 divider를 표시해주세요.

### ✅ 컨텐츠

- [x] 타이틀은 최대 2줄로 처리해주세요.
- [x] 컨텐츠를 클릭하면 해당 문서 URL로 이동해야 합니다.
- [x] 컨텐츠 검색 API 요청에 `from` 값을 넣어 무한 스크롤을 구현해주세요.

### ✅ Thumbnail / Favicon

- [x] favicon 로딩 중 에러가 발생하면 default favicon으로 대체해주세요.
- [x] thumbnail 로딩 중 에러가 발생하면 default thumbnail으로 대체해주세요.

### ✅ 북마크

- [x] 북마크 버튼을 통해 문서를 북마크하면(`save`),
  - [x] saved 상태의 버튼으로 전환하고,
  - [x] `200` 응답에서는 saved 상태의 버튼을 유지해주세요. (새로 고침, 다음 검색 시에는 저장 상태가 초기화되어도 좋습니다)
  - [x] 에러 응답에서는 save 상태의 버튼으로 전환하고 에러 모달을 띄워주세요.
- [x] 북마크 버튼을 통해 북마크를 취소하면(`unsave`),
  - [x] save 상태의 버튼으로 전환하고,
  - [x] `200` 응답에서는 save 상태의 버튼을 유지해주세요. (새로 고침, 다음 검색 시에는 저장 상태가 초기화되어도 좋습니다)
  - [x] 에러 응답에서는 saved 상태의 버튼으로 전환하고 에러 모달을 띄워주세요.

## 3️⃣ 회원 가입 (Sign Up)

### ✅ 기능 구현

- [ ] 회원 가입 페이지를 구현하세요.
- [ ] 입력 정보: 이름, 이메일, 휴대폰 번호, 비밀번호, 회원 유형(수강생/강사)
- [ ] 비밀번호 유효성 검사: 6~10자, 영문 소/대문자/숫자 중 2가지 이상 조합
- [ ] 가입 완료 시 강의 목록으로 이동하세요.

## 4️⃣ 강의 개설 (Create Course)

### ✅ 기능 구현

- [ ] 강의 개설 페이지를 구현하세요.
- [ ] 강의 목록 화면에서 '강의 개설' 버튼을 통해 진입하세요.
- [ ] 입력 정보: 강의명, 최대 수강 인원, 가격
- [ ] 등록 완료 시 강의 목록 화면으로 이동하세요.
- [ ] 수강생 회원은 강의를 등록할 수 없도록 제한하세요.

## 5️⃣ 수강 신청 (Course Registration)

### ✅ 강의 목록

- [ ] 강의 목록을 무한 스크롤로 구현하세요 (모바일 기준).
- [ ] 정렬 기능 구현: 최신 등록순, 신청자 많은 순, 신청률 높은 순
  - 신청률 = 신청자 수 / 최대 수강 인원
- [ ] 강의 아이템 표시 정보:
  - 강의명, 가격, 강사명, 현재 신청자 수, 최대 수강 인원

### ✅ 신청 기능

- [ ] 원하는 강의를 선택하여 신청할 수 있도록 구현하세요.
- [ ] 여러 강의를 한 번에 선택하여 신청 가능해야 합니다.
- [ ] 강사 회원도 수강 신청이 가능해야 합니다.

## 6️⃣ 로그인 (Login)

### ✅ 기능 구현

- [ ] 로그인 페이지를 구현하세요.
- [ ] 입력 정보: ID(이메일), 비밀번호
- [ ] 로그인 버튼 클릭 시 로그인이 완료되고 메인(혹은 강의 목록)으로 이동하세요.

## 7️⃣ Implementation Requirements

### ✅ 기술 스택 및 환경

- [ ] 모바일 웹 기준(768px) UI 구현 (shadcn UI 활용)
- [ ] Next.js App Router, SSR 아키텍쳐
- [ ] Semantic HTML 준수
- [ ] 테스트 코드 작성 (단일 비즈니스 로직, 컴포넌트, E2E)
